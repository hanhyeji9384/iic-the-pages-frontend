# IIC Welfare Project Rules

## 1. 커밋 메시지 규칙
- 모든 Git 커밋 메시지는 **한국어**로 작성합니다.
- 예시: `feat: 숙소 예약 캘린더 디자인 개선`, `fix: 공지사항 상세 페이지 레이아웃 수정`

## 2. 디자인 및 컴포넌트 원칙
- 모든 상세 페이지는 `SubpageLayout` 컴포넌트를 기반으로 구현하여 일관성을 유지합니다.
- 디자인은 항상 **프리미엄(Premium)하고 모던한 감성**을 지향합니다.
    - `rounded-[32px]` 또는 `rounded-[40px]`와 같은 큰 곡률의 테두리를 활용합니다.
    - `bg-[#f8f9fb]` 배경과 `white` 카드의 조합, 은은한 `shadow-sm` 또는 `shadow-black/5`를 사용하여 깊이감을 줍니다.
    - `animate-in`, `fade-in`, `slide-in` 등의 애니메이션 효과를 적절히 활용하여 생동감을 부여합니다.

## 3. 아이콘 사용 가이드
- 아이콘 라이브러리는 **`lucide-react`**를 기본으로 사용합니다.
- 각 페이지와 기능의 의미에 맞는 직관적인 아이콘을 배치합니다.

## 4. 라이브러리 및 스택
- 스타일링: Tailwind CSS (Vanilla CSS 스타일 선호)
- 상태 관리: React Hook (`useState`, `useEffect` 등)
- 데이터: `src/app/data/` 경로의 정의된 정적 데이터를 활용합니다.

## 5. 코드 품질 및 협업
- 가능한 한 깨끗하고 가독성 좋은 코드를 작성합니다.
- 중복되는 레이아웃이나 스타일은 컴포넌트화하여 공통으로 관리합니다.

## 6. 한국어 주석 규칙
- 모든 코드에 **비전공자도 이해할 수 있는 한국어 주석**을 작성합니다.
- 주석은 "무엇을 하는 코드인지"를 쉬운 말로 설명합니다. 전문 용어는 피하거나 괄호로 풀어씁니다.
- 다음 단위마다 주석을 답니다:
  - **컴포넌트 상단**: 이 컴포넌트(화면 조각)가 어떤 역할을 하는지 한 줄 설명
  - **상태(state) 선언부**: 각 상태 변수가 무엇을 저장하는지 설명
  - **주요 함수/핸들러**: 어떤 동작을 처리하는 함수인지 설명
  - **조건 분기/반복문**: 왜 이 조건이 필요한지 설명
  - **복잡한 계산/로직**: 계산 과정을 단계별로 설명
